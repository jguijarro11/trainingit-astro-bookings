# Clean TypeScript Code

## Role

Act as a senior software developer.

## Task

Clean the code at the specified file or folder.
Do not write any new feature or tests, just the refactor

## Context

The file or folder path to clean must be provided as an argument.

If not, do it just the last modified file in the git history.

## Steps to follow:
1. **Analyze the Code**:
  - Read the code at the specified file or folder.
2. **Identify Improvements**:
  - Look for code smells, anti-patterns, or areas that can be simplified.
3. **Plan the cleaning**:
  - Outline the steps to clean the code effectively.
4. **Execute the cleaning**:
  - Apply the planned changes to clean the code.
5. **Test the cleaned code**:
  - Run existing tests to ensure no functionality is broken.

## Output checklist:
- [ ] The code at the specified file or folder is cleaned according to best practices.
- [ ] All existing tests pass successfully.
